## 🎯 **ANSWERS TO YOUR QUESTIONS**

### **Q1: Most reliable way to fire Day One popup?**
**Answer:** `on_startup` with `days = 1` delay

**Why:**
- `on_game_start` doesn't exist in HOI4
- `on_startup` fires when game initializes (not on save reload)
- `on_daily_pulse` is inefficient and fires constantly
- Delayed event in country history works but is less flexible

**Best Practice:**
```
on_startup = {
    effect = {
        every_country = {
            limit = { is_ai = no }
            country_event = { id = X days = 1 }
        }
    }
}
```

---

### **Q2: How to scope event to human player only?**
**Answer:** Use TWO filters:

**Filter 1: In on_actions**
```
every_country = {
    limit = { is_ai = no }  # Only fire for human
    country_event = { id = X }
}
```

**Filter 2: In event trigger**
```
trigger = {
    is_ai = no
}
```

**Why both?**
- on_actions filter prevents unnecessary event calls
- Event trigger is backup safety check
- Prevents AI performance issues

---

### **Q3: Common mistakes preventing on_actions events?**

❌ **Mistake 1:** Using `on_game_start` (doesn't exist)
✅ **Fix:** Use `on_startup`

❌ **Mistake 2:** Not including `is_triggered_only = yes`
✅ **Fix:** Always add this to events called by on_actions

❌ **Mistake 3:** Localization key mismatch
✅ **Fix:** Ensure loc keys match exactly: `event_id.1.t`, `.d`, `.a`

❌ **Mistake 4:** Missing UTF-8 BOM in .yml files
✅ **Fix:** Always save .yml as UTF-8 with BOM

❌ **Mistake 5:** Event fires but AI receives it
✅ **Fix:** Add `is_ai = no` check

❌ **Mistake 6:** Multiple on_actions files override each other
✅ **Fix:** Use unique filenames, check for conflicts

---

### **Q4: Correct event trigger syntax?**

**Wrong:**
```
country_event = { id = X }  # Missing 'effect ='
```

**Correct for on_actions:**
```
effect = {
    country_event = { id = X }
}
```

**Correct for within events:**
```
immediate = {
    country_event = { id = X }
}
```

---

### **Q5: every_country vs ROOT in on_actions?**

**Use `every_country` when:**
- You want ALL countries to potentially receive event
- Combined with limit = { is_ai = no } for player-only

**Use `ROOT` when:**
- on_actions already scoped to specific country
- Example: `on_country_released` is already country-scoped

**For Day One popup:**
✅ Use `every_country` with player filter

---

### **Q6: Multiple on_actions files conflicts?**

**Yes, they can conflict!**

**How HOI4 handles on_actions:**
- All on_actions files are merged
- If same trigger appears multiple times, ALL fire
- Last file loaded can override if syntax conflicts

**Best Practice:**
- Use unique on_actions filenames
- Use descriptive trigger names
- Check vanilla files for conflicts
- Test thoroughly

## 🌍 **GLOBAL CRISIS SYSTEM - DESIGN ANSWERS**

### **Q1: Best way to track global escalation variable?**

**Answer:** Global variables (what this implementation uses)

**Comparison:**

| Method | Pros | Cons | Recommended? |
|--------|------|------|--------------|
| **Global variable** | Persistent across saves, easy to access, single source of truth | Requires 1.12+ | ✅ **YES** |
| State variable | Easy to implement | Not truly global, arbitrary state needed | ❌ No |
| Country variable | Works in older versions | Tied to specific country, can be lost | ❌ No |
| Scripted flags | Simple | Binary only, can't track levels | ❌ No |

**This implementation uses:**
```
set_variable = { global.tlww_crisis_level = 0 }
add_to_variable = { global.tlww_crisis_level = 5 }
```

---

### **Q2: How to implement monthly pulse without performance issues?**

**Answer:** Self-perpetuating hidden global_event

**This implementation:**
```
global_event = {
    id = tlww_crisis_pulse.1
    days = 30
}
```

**Inside the pulse event:**
```
immediate = {
    # Do all crisis checks
    
    # Schedule next pulse
    global_event = {
        id = tlww_crisis_pulse.1
        days = 30
    }
}
```

**Performance optimization:**
- Use `hidden = yes` (no UI overhead)
- Use `global_event` not `country_event` (single fire)
- Use `random_list` for probability (efficient)
- Minimize scope changes

## 🎮 **HOW THE SYSTEM WORKS**

### **On Game Start:**
1. `on_startup` fires
2. Initializes crisis system (sets variables to 0)
3. Schedules first monthly pulse (30 days)
4. Fires Day One popup for player (1 day delay)

### **Every Month:**
1. Hidden pulse event fires
2. Random chance checks for each flashpoint:
   - Taiwan: 10% monthly
   - Eastern Europe: 8% monthly
   - Panama: 5% monthly
   - Arctic: 6% monthly
   - Africa: 7% monthly
   - South America: 5% monthly
3. Each escalation adds to global crisis level
4. System checks if thresholds crossed:
   - 25%: "Rising" event fires
   - 50%: "Elevated" event fires
   - 75%: "Critical" event fires
   - 100%: "Last World War" event fires
5. Pulse schedules itself for next month

### **Player Interaction:**
- Decision available: "Monitor Global Crisis Level"
- Shows current crisis % and all flashpoint values
- Player can see escalation happening in real-time

## 🐛 **TROUBLESHOOTING**

### **Day One Event Not Firing:**

**Check 1:** Is on_actions in right location?
- Must be: `common/on_actions/tlww_on_actions.txt`

**Check 2:** Is localization file UTF-8 BOM?
- Re-save as UTF-8 with BOM

**Check 3:** Are loc keys correct?
- Event ID: `tlww_intro.1`
- Keys: `.1.t`, `.1.d`, `.1.a`

**Check 4:** Open console and type:
```
event tlww_intro.1
```
If event fires manually → on_actions issue
If event doesn't fire → event file issue

---

### **Crisis System Not Escalating:**

**Check 1:** Open console, type `tdebug`
- Look for `global.tlww_crisis_level` in variable list
- If not present → init didn't fire

**Check 2:** Type in console:
```
add_to_variable = { global.tlww_crisis_level = 5 }
```
Check if variable increases

**Check 3:** Check error.log for:
- "Unknown scripted effect"
- "Variable not found"

---

### **Events Firing for AI:**

**Fix:** Add to event:
```
trigger = {
    is_ai = no
}
```

---

### **Multiple Threshold Events Firing:**

**Cause:** Flags not being set properly

**Fix:** Each threshold event should set flag:
```
set_global_flag = tlww_crisis_low
```

And check for it:
```
NOT = { has_global_flag = tlww_crisis_low }
```
