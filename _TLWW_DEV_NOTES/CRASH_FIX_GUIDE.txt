## 🚨 **FATAL ERRORS FIXED (IN PRIORITY ORDER)**

### **1. SCRIPTED EFFECTS PARAMETERS (MOST CRITICAL)**

❌ **WRONG SYNTAX:**
```
tlww_add_crisis_level = {
    add_to_variable = { global.tlww_crisis_level = $amount$ }
}

# Called like:
tlww_add_crisis_level = { amount = 5 }  # DOES NOT WORK IN HOI4
```

✅ **CORRECT SYNTAX:**
```
# Define separate effects for each value
tlww_add_crisis_small = {
    add_to_variable = { global.tlww_crisis_level = 2 }
}

tlww_add_crisis_medium = {
    add_to_variable = { global.tlww_crisis_level = 3 }
}

# Called like:
tlww_add_crisis_medium = yes  # WORKS
```

**Why:**
- HOI4 scripted effects DO NOT support named parameters like `{ amount = 5 }`
- The `$variable$` syntax only works in scripted localisation
- Solution: Create separate effects or use pre-set variable values

---

### **2. NO `global_event` EFFECT (SECOND CRITICAL)**

❌ **WRONG:**
```
global_event = {
    id = tlww_crisis_pulse.1
    days = 30
}
```

✅ **CORRECT:**
```
USA = {
    country_event = {
        id = tlww_crisis_pulse.1
        days = 30
    }
}
```

**Why:**
- `global_event` does not exist in HOI4 1.17.4
- Use `country_event` scoped to a specific country
- USA is chosen because it always exists in your scenario

---

### **3. HISTORY FILE LEADER SYNTAX**

❌ **WRONG:**
```
set_country_leader = {  # This is an EFFECT, not history syntax
    character = USA_jd_vance
}
```

✅ **CORRECT (Character System):**
```
recruit_character = USA_jd_vance
```

✅ **CORRECT (Legacy Method):**
```
create_country_leader = {
    name = "J. D. Vance"
    picture = "gfx/leaders/USA/jd_vance.dds"
    ideology = conservatism
}
```

**Why:**
- `set_country_leader` is a scripted effect, not a history command
- History files use `recruit_character` or `create_country_leader`
- Effects like `set_country_leader` only work inside effect blocks in events

---

### **4. ADVISOR TRAIT DEFINITION**

❌ **WRONG:**
```
traits = { digital_strategist }  # ERROR if trait not defined
```

✅ **CORRECT:**
```
# Step 1: Define in common/country_leader/00_tlww_traits.txt
advisor_traits = {
    digital_strategist = {
        slot = political_advisor
        research_speed_factor = 0.05
        political_power_gain = -0.05
    }
}

# Step 2: Use in character/idea
traits = { digital_strategist }  # NOW WORKS
```

**Why:**
- All traits must be defined before use
- Advisor traits go in `common/country_leader/` NOT `common/ideas/`
- The trait definition contains the modifiers

## 🎯 **NON-CRASH ISSUES FIXED**

### **5. THIEL UNFIREABLE MECHANIC**

✅ **CORRECT IMPLEMENTATION:**

**In `common/characters/USA.txt`:**
```
USA_peter_thiel = {
    advisor = {
        can_be_fired = no
        
        available = {
            has_country_leader = {
                character = USA_jd_vance
                ruling_only = yes
            }
        }
    }
}
```

**In `common/ideas/00_tlww_ideas.txt`:**
```
political_advisor = {
    USA_peter_thiel = {
        allowed_to_remove = {
            NOT = {
                has_country_leader = {
                    character = USA_jd_vance
                    ruling_only = yes
                }
            }
        }
    }
}
```

**In `history/countries/USA - United States.txt`:**
```
add_ideas = { USA_peter_thiel }  # Auto-hire on start
```

**How it works:**
- `can_be_fired = no` provides basic protection
- `allowed_to_remove` check makes dismiss button appear ONLY when Vance is gone
- Auto-hired via history file
- Appears in advisor list immediately

## 🐛 **COMMON MISTAKES TO AVOID**

### **Mistake 1: Scripted Effect Parameters**
```
# NEVER DO THIS:
my_effect = {
    some_command = $param$  # DOES NOT WORK
}

# INSTEAD DO THIS:
my_effect_5 = {
    some_command = 5
}

my_effect_10 = {
    some_command = 10
}
```

### **Mistake 2: Using Effects in History**
```
# History files are NOT effect blocks
# WRONG:
set_country_leader = { }
add_to_variable = { }
if = { }

# CORRECT:
recruit_character = X
create_country_leader = { }
add_ideas = { }
set_politics = { }
```

### **Mistake 3: Undefined Traits**
```
# Always define before use
# 1. Create trait in common/country_leader/
# 2. Then use in character/idea
```

### **Mistake 4: Assuming vanilla effects exist**
```
# These DO NOT EXIST in HOI4:
global_event = { }         # Use country_event scoped to a country
set_party_leader = { }     # Use create_country_leader
add_crisis = { }           # Must define yourself
```

## ⚠️ **MINIMAL OVERRIDE DOCTRINE**

### **Principles:**
1. **Prefix all custom files:** `00_tlww_`, `tlww_`
2. **Never edit vanilla files directly**
3. **Use scripted effects for reusable logic**
4. **Namespace all events:** `tlww_category.id`
5. **Test incrementally:** Add one system at a time

### **Safe Override Patterns:**

✅ **SAFE:**
```
common/ideologies/00_ideologies.txt         # Full replacement OK
localisation/english/tlww_*.yml             # Only adds, no conflicts
common/scripted_effects/tlww_effects.txt    # Only adds
events/tlww_*.txt                           # Only adds
```

⚠️ **CAREFUL:**
```
common/on_actions/tlww_on_actions.txt       # Merges with vanilla
history/countries/USA - United States.txt   # Replaces vanilla USA
```

❌ **DANGEROUS:**
```
common/defines/00_defines.lua               # Global changes
common/ai_strategy/default.txt              # Breaks AI
interface/*.gfx                             # Can cause UI crashes
```

## 🎯 **PERFORMANCE TIPS**

### **Crisis Pulse Optimization:**
```
# GOOD: Single hidden event
USA = {
    country_event = {
        id = tlww_crisis_pulse.1
        days = 30
    }
}

# BAD: Every country fires
every_country = {
    country_event = {
        id = tlww_crisis_pulse.1
    }
}
```

### **Variable Usage:**
```
# GOOD: Global variables
set_variable = { global.crisis = 50 }

# BAD: Country variables for global state
set_variable = { crisis = 50 }  # Only in this country's scope
```

### **Event Scoping:**
```
# GOOD: Direct scope
USA = { country_event = { id = X } }

# BAD: Iterating to find
every_country = {
    limit = { tag = USA }
    country_event = { id = X }
}
```
